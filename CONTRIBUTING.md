# Mithelfen

Ich versuche diese Software mit Hilfe von Codex (GPT-5.6-Sol aktuell) zu entwickeln.
Ein Großteil der Codebase wurde mit Codex generiert.

Gerne können Sie bei der Weiterentwicklung mithelfen. Möglichkeiten sind unter anderem fachliche Anforderungen zu kommunizieren und Bugs zu melden (via GitHub Issues) oder Features zu implementieren oder Bugs zu fixen (via GitHub Pull Requests). Dabei können auch Frontier-LLMs verwendet werden (GPT-5.6-Sol oder andere ähnlich starke Modelle).

## Mein Entwicklungsprozess

Ich delegiere die Entwicklung großteils an Codex. Ich teste die implementiere meistens End-to-End manuell.

## PRs

Ich bitte bei Features, dass der PR entweder selbst End-to-End getestet wurde oder Playwright E2E-Tests enthält, die zeigen, dass die Funktionalität des PRs End-to-End funktioniert.
Das macht für mich das Reviewen kosten- und aufwandseffizienter.

Grundsätzlich sollen Anforderungen automatisierte Tests haben.

Kurz zu mir, damit Sie wissen, mit wem Sie kommunizieren: Ich habe über zehn Jahre Erfahrung in der Webentwicklung. Dementsprechend kenne ich mich fachlich gut in dem Bereich aus. Im Bereich Buchhaltung habe ich eine zweijährige kaufmännische Ausbildung als kaufmännischer Assistent und mache die doppelte Buchführung seit ungefähr vier Jahren selber.
